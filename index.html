Some reasons to use Symfony:

  • Staying Power

    • It was created by Sensio Labs, and they've been doin this stuff for like, a long time. Fine, more than 12 years. So they've got this.

Some reasons to use Symfony:

  • Reputation/References

    • Yahoo!, Dailymotion, Opensky.com, Exercise.com and even applications such as phpBB and Drupal. And soon Franklin American Mortgage Company will join this list...

Some reasons to use Symfony:

  • Interoperability

    • Symfony has mad respect for "de facto" standards of PHP: PHPUnit, naming conventions, code style. Framework creator Fabien Potencier is even a voting member on the PHP Framework Interoperability Group check it!
    • The framework itself is built via core components that can even be pulled out and used individually or in smaller groups. One popular example is the micro-framework Silex, built by Sensio Labs via these components.
    • Drupal 8 is slated to adopt many core Symfony2 core components such as HttpFoundation, Routing, and ClassLoader.

Some reasons to use Symfony:

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This let's you apply broader style changes, like switching the background.

"blackout"

"soothe"

Custom Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 

Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );
