Super awesome MVC framework for PHP

  1. Symfony is an Open Source distributed PHP framework.
  2. You don't need a framework to develop in PHP, but they sure make life easier.

Some reasons to use Symfony:

  • Staying Power

    • It was created by Sensio Labs, and they've been doin this stuff for like, a long time - more than 12 years. So they've got this.

Some reasons to use Symfony:

  • Reputation/References

    • Yahoo!, Dailymotion, Opensky.com, Exercise.com and even applications such as phpBB, Drupal, and now Franklin American Mortgage Company has joined the list...

Some reasons to use Symfony:

  • Interoperability

    • Sensio Labs is instrumental in PHP standards (PHPUnit, naming conventions, code style). Creator Fabien Potencier is a voting member on the PHP Framework Interoperability Group
    • The framework is built via core components that can even be pulled out and used individually or in smaller groups. One popular example is the micro-framework Silex, built by Sensio Labs via these components.
    • Drupal 8 is slated to adopt many core Symfony2 core components such as HttpFoundation, Routing, and ClassLoader.

Some reasons to use Symfony:

  • Resources

    • Fear not! You aren't alone in your Symfony development. Symfony has a strong developer community, including some right here in Nashville.

What is a Service?

... a Service is any PHP object that performs some sort of "global" task. It's a purposefully-generic name used in computer science to describe an object that's created for a specific purpose (e.g. delivering emails). Each service is used throughout your application whenever you need the specific functionality it provides.

Digging A Little Deeper

Controller

  • ~Action Functions - must end with Action
  • Annotations
    • @Route - path and route name
    • @Method - DELETE/GET/POST/PUT
    • @Template - define default template to use

Digging A Little Deeper

Controller - Route

  • Sensio\Bundle\FrameworkExtraBundle\Configuration\Route
  • Action Level: 
    
            /**
             * @Route("/", name="page_index")
             *  // or
             * @Route("/edit/{id}", name="page_edit")
             */
            public function editAction($id)
  • Controller Level: 
    
            /**
             * @Route("/page")
             */
            class PageController extends Controller

Digging A Little Deeper

Controller - Method

  • Sensio\Bundle\FrameworkExtraBundle\Configuration\Method
  • public function indexAction(Request $request)
  • Example: 
    
            /**
             * @Method("GET")
             *  // or
             * @Method({"GET", "POST"})
             */

Digging A Little Deeper

Controller - Template

  • Sensio\Bundle\FrameworkExtraBundle\Configuration\Template
  • Example:
    
             /**
              * @Template("famcDemoBundle:Page:show.html.twig")
              */
              public function editAction($id)

Digging A Little Deeper

Controller - Request

  • Symfony\Component\HttpFoundation\Request
  • Wraps up all the $_SERVER, $_GET, $_POST, $_FILES, $_COOKIE in a nice neat package.
  • Includes Session Data.
  •  function indexAction(Request $request)

Digging A Little Deeper

Controller - Response

  • Symfony\Component\HttpFoundation\Response
  • Nice OO way to manage header content
  • Layer of abstraction sitting between browser and controller
  • Makes controller agnostic of rendered content
